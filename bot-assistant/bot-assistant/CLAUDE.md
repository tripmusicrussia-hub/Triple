# Проект: Triple Bot

Telegram-бот для продуктового магазина. Основная функция — фотографировать накладные от поставщиков и автоматически создавать черновики приходов в системе учёта Sigma.

## Архитектура

**Стек:**
- `python-telegram-bot 21.5` — бот
- Groq Vision API (`meta-llama/llama-4-scout-17b-16e-instruct`) — распознавание накладных с фото
- Sigma API (`api-s07.sigma.ru`) — система учёта товаров/приходов
- Render — деплой (автодеплой с GitHub при пуше в `main`)
- GitHub: `https://github.com/tripmusicrussia-hub/Triple`

**Поток обработки накладной:**
1. Пользователь (только ADMIN_ID) отправляет фото накладной
2. `recognize_invoice()` в `sigma_api.py` → Groq Vision → JSON с товарами
3. Бот показывает список товаров с кнопками редактирования
4. Пользователь правит ошибки, затем жмёт "Загрузить в Sigma"
5. `process_invoice()` → ищет/создаёт товары в Sigma → создаёт DRAFT приход

## Ключевые файлы

| Файл | Назначение |
|------|-----------|
| `bot.py` | Главный обработчик бота, инлайн-клавиатуры, flow накладной |
| `sigma_api.py` | SigmaAPI класс, распознавание, матчинг товаров, создание прихода |

## Сервисы и ключи

Все ключи в env vars на Render (не хардкодить в коде):
- `BOT_TOKEN` — Telegram бот
- `ADMIN_ID` — только этот пользователь работает с накладными
- `GROQ_API_KEY` — Groq Vision для OCR накладных
- `SIGMA_LOGIN` / `SIGMA_PASSWORD` — логин в Sigma (`+79174854325` / `IfrbIfrb680`)

## Sigma API

- Base URL: `https://api-s07.sigma.ru`
- Авторизация: сначала `/oauth/token` (password grant) → получаем Bearer token
- Приход создаётся как DRAFT (статус `"DRAFT"`) — пользователь проводит его вручную в Sigma
- Кэш товаров (`_products_cache`) — глобальный, загружается при первом использовании, обновляется если количество товаров изменилось

## Матчинг товаров (`find_product_in_cache`)

Нечёткий поиск по токенам + обязательные числа:
- Числа **≥ 20** в названии накладной — **обязательны** для совпадения (вес упаковки, объём)
- **% жирности** (3.2%, 9%, 15% и т.д.) — если есть в обоих, должны **точно совпадать** (иначе матчились бы Молоко 3.2% → Молоко 2.5%)
- Порог совпадения токенов: 35%

### Словари замен

**`ABBREVIATIONS`** — расшифровки аббревиатур (накладная → Sigma):
- `овк` → `очень важная корова`
- `мп` → `молочный переулок`

**`PRODUCT_NAME_FIXES`** — явные замены несовпадающих названий:
- `васькино счастье` / `вас. счас.` → `катык зеленодольский по домашнему` (в Sigma хранится под другим именем)
- `карламан какао 7.5` → `карламан какао 7`

### Если товар не найден
Бот **создаёт новый товар** в Sigma через `create_product()` — категория определяется автоматически по `CATEGORY_KEYWORDS`.

## Накладные

Бот работает преимущественно с **печатными накладными** (расходные накладные от поставщиков молочки). Рукописные накладные пока не в приоритете — свободные модели плохо распознают мелкие рукописные цифры.

Основной поставщик: **Молочный переулок** (ИП/ООО), накладная содержит колонки: Наименование / Заказ / Цена / Сумма.
